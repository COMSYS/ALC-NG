# ALC-NG

The project contains the sanitization tool **ALC-NG** to clean latex projects. It strips all unused files, comments, if statements and more from the project that is not required to generate the final pdf. The project is inspired by the  Google-maintained [`arxiv-latex-cleaner`](https://github.com/google-research/arxiv-latex-cleaner).

## Reliable Analysis of Unused Content

- The tool requires a working latex installation to compile the document and determine unused files.
- We use [Tree-Sitter](https://tree-sitter.github.io/tree-sitter/) to parse each latex file and strip unused parts
- The implementation has been tested on over XX arxiv papers with a success rate of XX

## Remove Sensitive Information

- We use exiftool to strip metadata from images and pdfs.

## Optimize Size

- The tool allows to resize images to a predefined size and format
- Unused content is removed
- Per default we only keep the generated .bbl file for bibliographies

## Validation

- The tool can make a pixel-perfect comparison between the original and cleaned version to ensure that it did not break the project.

## arXiv ready

- The cleaned project can directly be packaged into a `.tar.gz` archive, ready to upload to arXiv.

# Usage

# TL;DR Docker
Assuming a working [Docker](https://www.docker.com/) install, you can run the following in this folder to build a Docker Image:
```bash
docker build -t alc_ng .
```

## Local

Using this image, you can run all tooling in this repository. To simply clean a latex project on your local disk, use:
```bash
docker run --rm -v <path-to-project>:/input -v <desired-output-path>:/output alc_ng /input /output --force
```
If you also want to compare the cleaned pdfs to pdfs generated by the original input, use the `-c` or `--compare` flags like this:
```bash
docker run --rm -v <path-to-project>:/input -v <desired-output-path>:/output alc_ng  /input /output --force --compare
```
This will compare each page pixel-by-pixel and produce a report on differences.

## Examples

To run the cleaner on the example shown in our Appendix, run the following command.
```bash
docker run --rm alc_ng --compare /examples/appendix
```
We also supplied another arxiv submission (`1806.01078`) which can be tested like this:
```bash
docker run --rm alc_ng --compare /examples/1806_01078v1
```

### Output of cleaned example
To obtain the cleaned output of an example, do the following:
```bash
docker run --rm -v <output-path>:/output alc_ng --compare /examples/<example> /output
```

## Arxiv Submission
Otherwise, you can validate the cleaner on any arxiv id. The cleaner will automatically download the source code of the latest version and run itself.
```bash
docker run --rm --entrypoint ./arxiv_dl alc_ng --compare --force <ARXIV_ID>
```

### Output of cleaned example
To obtain the cleaned output of an example, do the following:
```bash
docker run --rm --entrypoint ./arxiv_dl alc_ng --compare --force <ARXIV_ID> --tar - > output.tar
```
This will create a tar file in the current folder with the cleaned output.

# Main Features

## Cleaning

* Removes all files not strictly needed to compile the resulting pdf
* Supports multiple main files
* Removes line comments, comment environments, custom commands, `\iffalse` comments and much more
* Preserves comment-like syntac in verbatim environments
* Optionally removes metadata from files (`--strip-exif` flag)
* Only preserves `.bbl` files and not `.bib` files (`.bib` files can be kept using the `--keep-bib` flag)


## Ease-Of-Use

* Creates a ready-to-submit gzipped tar archive (use `--tar <filename>.tar.gz` or `--tar -` to pipe the archive to stdout)

# Usage

## `cleaner`
The primary tool for cleaning a single submission folder. You can see a full list of options by running `cleaner -h`.

```bash
cleaner [INPUT_PATH] [OUTPUT_PATH] \ # Optional, defaults are ./input and ./cleaned
  --strip-exif \ # Run exiftool and remove metadata from images and pdfs. This can break pixel-perfect comparisons.
  --keep-bib \ # Copy the .bib file when cleaning. The .bbl file is always copied and suffices for compilation.
  --force \ # Continue cleaning, despite hitting errors. Most results will still work, despite hitting errors.
  --verbose \ # Enable verbose logging.
  --compare \ # Render the original and cleaned pdf as images, compare them pixel-by-pixel and print a report.
  --latex-cmd \ # Change default latex binary.
  --latex-args \ # Add additional arguments to be passed to the latex binary.
  --exiftool-cmd \ # Change default exiftool binary.
  --exiftool-cmd \ # Add additional arguments to be passed to the exiftool binary.
  --resize-image \ # Resize images to the size specified by --im-size.
  --im-size \ # Change sizes for images.
  --clean-classes \ # Also clean .sty, .cls and other latex class files.
  --no-zzrm \ # Ignore statements in a 00Readme file that is present.
  --tar <PATH> \ # Create a gzipped tar archive ready for upload. The <PATH> can be -, then the archive will be piped to stdout.
```

## `arxiv_dl`
Similar to cleaner, but instead of providing a local folder, you can supply an arxiv submission id and the tool will download the source and run the cleaner.  You can see a full list of options by running `cleaner -h`.
```bash
arxiv_dl <ARXIV_ID> \ # Define id for cleaning e.g. 2303.12712
  --strip-exif \ # Run exiftool and remove metadata from images and pdfs. This can break pixel-perfect comparisons.
  --keep-bib \ # Copy the .bib file when cleaning. The .bbl file is always copied and suffices for compilation.
  --force \ # Continue cleaning, despite hitting errors. Most results will still work, despite hitting errors.
  --verbose \ # Enable verbose logging.
  --compare \ # Render the original and cleaned pdf as images, compare them pixel-by-pixel and print a report.
  --latex-cmd \ # Change default latex binary.
  --latex-args \ # Add additional arguments to be passed to the latex binary.
  --exiftool-cmd \ # Change default exiftool binary.
  --exiftool-cmd \ # Add additional arguments to be passed to the exiftool binary.
  --resize-image \ # Resize images to the size specified by --im-size.
  --im-size \ # Change sizes for images.
  --clean-classes \ # Also clean .sty, .cls and other latex class files.
  --no-zzrm \ # Ignore statements in a 00Readme file that is present.
  --tar <PATH> \ # Create a gzipped tar archive ready for upload. The <PATH> can be -, then the archive will be piped to stdout.
```

# Local Installation

You need a working installation of Rust. You can install Rust using [rustup](https://rustup.rs/). As this cleaner needs to compile the latex project, a working latex installation is also needed.

## Prepare Tree Sitter
This step assumes a working nodejs installation.
```bash
cargo install tree-sitter-cli
```
```bash
cd external/ts-latex && make clean && tree-sitter generate && make
```

## Compile project itself
Go back to the root of the project.
```bash
cargo build
```
All binaries can then be found in `target/debug`.
```bash
./target/debug/cleaner <options>
./target/debug/arxiv_dl <options>
```

## Addtional optional dependencies
If you want to strip exif data using the cleaner, you need to install [Exiftool](https://exiftool.org/).
